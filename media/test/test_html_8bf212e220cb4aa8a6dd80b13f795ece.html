Software Testing Knowledge Quiz

填空题

1. Google's approach where developers write their own tests is called .

2. The testing technique that examines internal program structure is called testing.

3. The metric that shows how much of an application's code has tests is called .

选择题

1. Which of these was NOT a problem Google faced with GWS before implementing automated testing?

A. Releases were becoming buggier
B. Productivity had slowed dramatically
C. Team members had little confidence when making changes
D. The server hardware kept failing

2. Which testing level focuses on verifying interactions between components?

A. Unit testing
B. Integration testing
C. System testing
D. Acceptance testing

3. What percentage of branches need to be executed for 100% branch coverage?

A. 50%
B. 75%
C. 90%
D. 100%

4. Which testing technique divides input data into equivalence classes?

A. Boundary value analysis
B. Equivalence partitioning
C. Random testing
D. Exhaustive testing

5. Which type of testing is typically performed by business users?

A. Unit testing
B. Integration testing
C. User acceptance testing
D. Performance testing

答案解析

1. 答案：Developer Driven Testing (DDT)
解析：Google instituted a policy of developer-driven automated testing where developers write their own tests.

2. 答案：White-box
解析：White-box testing examines the internal structure and implementation of the software.

3. 答案：Code coverage
解析：Code coverage measures how much of the source code is executed during testing.

4. 答案：D. The server hardware kept failing
解析：The problems with GWS were related to software quality and development process, not hardware failures.

5. 答案：B. Integration testing
解析：Integration testing verifies the interactions between components through their interfaces.

6. 答案：D. 100%
解析：100% branch coverage requires all possible branches to be executed at least once.

7. 答案：B. Equivalence partitioning
解析：Equivalence partitioning divides input data into classes where test cases can be considered equivalent.

8. 答案：C. User acceptance testing
解析：UAT is performed by business users to verify the system meets their needs.
